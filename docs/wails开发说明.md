**技术栈总览**

| 类别     | 技术/库/工具          | 说明               
| -------- | --------------------- | ------------------------------------------------------------ |
| **后端** | **Go**                | 编程语言，用于构建后端 API 服务                              |
|          | **Gin**               | Go 的 Web 框架，用于处理 HTTP 请求和路由                     |
|          | **SQLite**            | 嵌入式关系型数据库，无需安装，易于部署，适合单机或小型应用   |
|          | **ORM (Gorm/Xorm)**   | 对象关系映射 (ORM) 库，用于简化数据库操作                    |
|          | **Swagger (OpenAPI)** | API 文档生成工具，自动生成 API 文档                          |
|          | **godotenv**          | 环境变量管理库，从 .env 文件加载环境变量                     |
|          | **Logrus/Zap**        | 日志库，用于记录应用程序运行时的各种信息                     |

**wails开发说明**
1. 这是一个大型的项目 为了方便维护，需要需要遵守以下规则
    1. 需要谨慎修改
    2. 需要有良好的注释
    3. 需要有良好的目录结构
    4. 需要有良好的代码规范、代码结构、代码风格
    5. 不要过度封装和设计，但是也不要过渡耦合
    6. 代码要简洁可读性要高
    7. 依赖库最好使用最佳实践
2. 后端
    1. 所有接口全部返回200状态码，在返回体中返回具体的业务状态码
